## Manual QA — 90-Day Plan

### The Most Important Thing to Understand First

You are not starting from zero. You have:
- Domain knowledge of how real software breaks
- Understanding of edge cases that automation misses
- Experience communicating quality risk to stakeholders
- Mental models of how users actually behave

None of this is worthless. The transition is about redirecting these skills — not replacing them.

### Phase 1 (Days 1–30): Stop Defending, Start Building

**Phase 1 Deliverable:** A public GitHub repo with 10 testRigor tests, an Applitools visual baseline, and one DeepEval LLM quality test. README explains the strategy behind each.

| Week | Daily Focus (1–2 hrs) | Weekend Project (4–5 hrs) |
|---|---|---|
| Week 1 | Install testRigor free account. Watch 2 tutorial videos. Write your first plain English test. | Convert your 5 most-run manual regression tests to testRigor. Document time saved. |
| Week 2 | Create Applitools free account. Run visual testing tutorial. | Build a visual regression baseline for a web app you test regularly. |
| Week 3 | Install DeepEval. Follow the quickstart. Write one LLM evaluation test. | Write 3 LLM evaluation tests using DeepEval against a free OpenAI API call. |
| Week 4 | Set up GitHub account. Learn basic Git: add, commit, push. | Push all your work to a public GitHub repo. Write a README. Post on LinkedIn. |

### Phase 2 (Days 31–60): Build the AI Layer

**Phase 2 Deliverable:** An AI Testing Framework design document (4–6 pages). Plus: you have mentored at least one junior QA on one new tool.

| Week | Daily Focus (1–2 hrs) | Weekend Project (4–5 hrs) |
|---|---|---|
| Week 5 | Learn RAGAS basics. Read the quickstart documentation. | Build one RAGAS evaluation pipeline for a domain-specific document Q&A system. |
| Week 6 | Study Playwright Python basics. Understand auto-wait and role locators. | Write 10 Playwright tests. Compare with your testRigor tests — document differences. |
| Week 7 | Research 5 GCC AI Quality job descriptions. Note exact skills listed. | Write your AI Testing Framework design document. Use the template in `05-interview-prep`. |
| Week 8 | Teach one junior QA how to use testRigor. Document what they learned. | Polish your GitHub profile and README. Share with one senior connection for feedback. |

### Phase 3 (Days 61–90): Reposition and Apply

**Phase 3 Deliverable:** 3 GCC applications submitted. LinkedIn profile rewritten. One informational call booked.

| Week | Daily Focus (1–2 hrs) | Weekend Project (4–5 hrs) |
|---|---|---|
| Week 9 | Rewrite LinkedIn headline and About section using templates in `06-linkedin-github`. | Complete LinkedIn rewrite. Update CV with outcome statements, not task descriptions. |
| Week 10 | Identify 10 specific GCC roles that match your domain + AI Quality skills. | Write 3 tailored cover paragraphs. Research each company's AI testing challenges. |
| Week 11 | Practice system design answer for QA architecture questions — out loud. | Submit 3 applications. Post one technical LinkedIn article about AI testing (200 words). |
| Week 12 | Practice STAR behavioural answers. Prepare salary negotiation response. | Book one informational call with a QA professional at a GCC. Apply to 2 more roles. |

### Daily Habits That Compound
- **15 min:** Read one article about AI testing, AI quality, or LLM evaluation. Save the key insight.
- **30 min:** Build something. Even one test case, one eval metric, one README update.
- **5 min:** LinkedIn. Post a small insight, comment on an AI testing post, connect with one GCC engineer.

## Selenium Engineer — 60-Day Upgrade

### Phase 1 (Days 1–30): Migrate Your Selenium Skills

**Phase 1 Deliverable:** A public GitHub repo with 20+ Playwright tests migrated from Selenium, running in CI/CD via GitHub Actions, with a README explaining the architectural differences.

| Week | Daily Focus | Weekend Project |
|---|---|---|
| Week 1 | Install Playwright Python. Run the official tutorial. Write your first 5 tests. | Migrate your 10 most-used Selenium test patterns to Playwright. Document the differences. |
| Week 2 | Learn Playwright locator strategies — role-based, text, placeholder. Practice each. | Rewrite migrated tests using role-based locators. Compare readability with original Selenium. |
| Week 3 | Set up GitHub Actions CI pipeline. Learn pytest fixtures. | Add fixtures for login state, test data, browser setup. Reduce test duplication. |
| Week 4 | Explore Playwright trace viewer. Run broken tests to see the trace output. | Push complete suite to GitHub. Write README with Selenium vs Playwright comparison. Post on LinkedIn. |

### Phase 2 (Days 31–60): Add the AI Layer

**Phase 2 Deliverable:** An AI-augmented test suite combining Playwright for UI + DeepEval for LLM quality evaluation.

| Week | Daily Focus | Weekend Project |
|---|---|---|
| Week 5 | Install GitHub Copilot. Practice writing tests from comments. Measure time savings. | Use Copilot to write 15 new Playwright tests for a domain-specific application. |
| Week 6 | Install DeepEval. Run the quickstart. Understand AnswerRelevancy and Hallucination metrics. | Write 5 DeepEval tests evaluating a public LLM API. Add to GitHub repo. |
| Week 7 | Install RAGAS. Understand faithfulness vs relevancy vs context precision. | Build one RAGAS evaluation pipeline. Add to portfolio. |
| Week 8 | Research 5 GCC job descriptions for AI Quality / SDET roles. List exact skills required. | Write your AI Testing Framework design document (4 pages). Share with one senior connection. |

## When You Feel Like Stopping

There will be a day — probably around Day 30 — when the progress feels invisible. This is normal.

Two things help:
1. Look at your GitHub contribution graph. You have been building for 30 days. That graph is your proof of motion.
2. Look at one job description for the role you are targeting. Read the salary. Remember why you started.

**Every AI Quality Architect currently working at a GCC was, at some point, a QA engineer with the same doubts you have tonight. The difference is they did not stop on Day 30.**
