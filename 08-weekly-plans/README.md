## Manual QA — 90-Day Plan

### The Most Important Thing to Understand First

You are not starting from zero. You have domain knowledge of how real software breaks, understanding of edge cases that automation misses, and experience communicating quality risk to stakeholders. The transition is about redirecting these skills — not replacing them.

### Phase 1 (Days 1–30): Stop Defending, Start Building

**Phase 1 Deliverable:** A public GitHub repo with 10 testRigor tests, an Applitools visual baseline, and one DeepEval LLM quality test. README explains the strategy behind each.

| Week | Daily Focus (1–2 hrs) | Weekend Project (4–5 hrs) |
|---|---|---|
| Week 1 | Install testRigor free account. Watch 2 tutorial videos. Write your first plain English test. | Convert your 5 most-run manual regression tests to testRigor. Document time saved. |
| Week 2 | Create Applitools free account. Run visual testing tutorial. | Build a visual regression baseline for a web app you test regularly. |
| Week 3 | Install DeepEval. Follow the quickstart. Write one LLM evaluation test. | Write 3 LLM evaluation tests using DeepEval against a free OpenAI API call. |
| Week 4 | Set up GitHub account. Learn basic Git: add, commit, push. | Push all your work to a public GitHub repo. Write a README. Post on LinkedIn. |

### Daily Habits That Compound
- **15 min:** Read one article about AI testing, AI quality, or LLM evaluation
- **30 min:** Build something — even one test case, one README update
- **5 min:** LinkedIn — comment on an AI testing post, connect with one GCC engineer

## Selenium Engineer — 60-Day Upgrade

### Phase 1 (Days 1–30): Migrate Your Selenium Skills

**Phase 1 Deliverable:** A public GitHub repo with 20+ Playwright tests migrated from Selenium, running in CI/CD via GitHub Actions, with a README explaining the architectural differences.

| Week | Daily Focus | Weekend Project |
|---|---|---|
| Week 1 | Install Playwright Python. Run the official tutorial. Write your first 5 tests. | Migrate your 10 most-used Selenium test patterns to Playwright. Document the differences. |
| Week 2 | Learn Playwright locator strategies — role-based, text, placeholder. Practice each. | Rewrite migrated tests using role-based locators. Compare readability with original Selenium. |
| Week 3 | Set up GitHub Actions CI pipeline. Learn pytest fixtures. | Add fixtures for login state, test data, browser setup. Reduce test duplication. |
| Week 4 | Explore Playwright trace viewer. Run broken tests to see the trace output. | Push complete suite to GitHub. Write README with Selenium vs Playwright comparison. Post on LinkedIn. |

## When You Feel Like Stopping

There will be a day — probably around Day 30 — when the progress feels invisible. This is normal.

Two things help:
1. Look at your GitHub contribution graph. You have been building for 30 days. That is your proof of motion.
2. Look at one job description for the role you are targeting. Read the salary. Remember why you started.

**Every AI Quality Architect currently working at a GCC was, at some point, a QA engineer with the same doubts you have tonight. The difference is they did not stop on Day 30.**
