# Learning Path — What to Learn and In What Order

## The Most Important Rule

**Build before you finish learning.**

Most engineers study for 3 months and build nothing. A working project with bugs teaches more than 20 tutorials. The GitHub repo is the credential — not the certificate.

---

## Learning Path by Current Role

### If You Are a Manual QA Engineer

**Month 1 — Stop defending, start building**

| Week | Learn | Build |
|---|---|---|
| 1 | testRigor basics — plain English automation | 10 testRigor tests for a web app you use |
| 2 | Applitools — visual regression concepts | Visual baseline + one detected change |
| 3 | DeepEval — LLM evaluation basics | First AI quality test with pass/fail threshold |
| 4 | GitHub Actions — CI/CD basics | All tests running in CI automatically |

**Month 2 — Build the AI layer**

| Week | Learn | Build |
|---|---|---|
| 5 | RAGAS — RAG pipeline evaluation | Domain-specific Q&A evaluation pipeline |
| 6 | AI Quality governance concepts | AI testing framework design document (draft) |
| 7 | GCC interview patterns for QA | 5 STAR stories with quantified outcomes |
| 8 | LinkedIn repositioning | Updated profile + first LinkedIn post about your build |

**Month 3 — Apply and position**

| Week | Learn | Build / Do |
|---|---|---|
| 9 | Playwright basics | Rewrite 5 manual tests as Playwright automated tests |
| 10 | System design for AI quality | Complete framework design document |
| 11 | Salary negotiation scripts | Submit 3 GCC applications |
| 12 | Interview practice | Book one informational call with a GCC quality engineer |

---

### If You Are a Selenium Automation Engineer

**Month 1 — Migrate your stack**

| Week | Learn | Build |
|---|---|---|
| 1 | Playwright Python — auto-wait, locators | Migrate 10 Selenium tests to Playwright |
| 2 | Playwright advanced — trace viewer, CI | Add GitHub Actions, document before/after comparison |
| 3 | GitHub Copilot — AI-assisted test writing | Use Copilot to write 15 new tests. Measure time saved. |
| 4 | Role-based locators — accessibility first | Rewrite tests using `get_by_role()` instead of XPath |

**Month 2 — Add the AI evaluation layer**

| Week | Learn | Build |
|---|---|---|
| 5 | DeepEval — installation and quickstart | 5 LLM evaluation tests for a public API |
| 6 | RAGAS — faithfulness and relevancy | One RAG evaluation pipeline |
| 7 | AI testing system design | Draft your framework design document |
| 8 | GCC targeting for Playwright roles | LinkedIn profile update + 3 job applications |

---

### If You Are an SDET / Quality Engineer

**Month 1 — Build the AI quality layer**

| Week | Learn | Build |
|---|---|---|
| 1 | DeepEval — full metric library | Comprehensive LLM evaluation suite for your domain |
| 2 | RAGAS — advanced metrics | RAG pipeline evaluation with threshold calibration |
| 3 | LangSmith — production monitoring | Tracing setup for a Spring AI or LangChain app |
| 4 | Statistical pass/fail design | Document your threshold strategy |

**Month 2 — Architect and position**

| Week | Learn | Build |
|---|---|---|
| 5 | AI Quality Architect interview patterns | System design answer practised out loud |
| 6 | Testcontainers — AI integration testing | Test suite for an AI-integrated Java/Python app |
| 7 | Governance and release sign-off design | Complete AI testing framework document |
| 8 | Senior GCC targeting | 5 applications to AI Quality / SDET roles |
